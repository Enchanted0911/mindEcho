# Mermaid 图表生成器

## 角色与原则

根据用户任务生成 Mermaid 图表代码：**先解析任务 → 按直观程度与表现形式选型 → 选组件与连线 → 按项目规范写语句 → 自检**。  
生成前必须阅读并遵循项目内的 **`./reference.md`**，其中包含语法规则、节点/连线示例、特殊字符处理与检查清单。

## 执行工作流

### 第一步：解析任务

从任务描述中提取：

1. **要表达什么**：业务流程、系统交互、状态流转、数据结构、时间排期等
2. **受众与用途**：技术设计、产品说明、会议汇报等，决定详细程度
3. **复杂度**：步骤/参与者/状态数量，决定是否拆子图、分多层

若不清晰，先向用户确认上述三点再继续。

### 第二步：选择图表类型（直观程度 + 表现形式）

按「要表达什么」和「怎样最直观」选一种主类型：

| 要表达的内容           | 推荐类型        | 声明/示例方向                    |
|------------------------|-----------------|----------------------------------|
| 步骤、分支、流程       | 流程图 Flowchart | `graph TD` / `graph LR`          |
| 多方调用顺序、接口交互 | 时序图 Sequence | `sequenceDiagram`                |
| 类、接口、关系         | 类图 Class      | `classDiagram`                   |
| 状态与状态迁移         | 状态图 State    | `stateDiagram-v2`                |
| 实体与表关系           | ER 图           | `erDiagram`                      |
| 时间线与排期           | 甘特图 Gantt    | `gantt`                          |

同一任务若有多角度，可生成多张图（如：总体流程 + 某环节时序图），在说明中注明各自用途。

### 第三步：选择组件与样式

在选定类型下，按 `./reference.md` 中的约定选组件，保证「表现形式」统一：

- **流程图**：开始/结束用圆角 `( )`，判断用菱形 `{ }`，输入/输出用平行四边形 `[/ /]`，普通步骤用矩形 `[ ]`；主流程用 `-->`，分支用 `-->|"标签"|`，次要/异步用 `-.->`。
- **时序图**：用 `participant X as 显示名`；同步用 `->>`，返回用 `-->>`，异步用 `-)`；需要时用 `alt/else/loop`。
- **状态图**：用 `[*]` 表示起止，迁移用 `状态 --> 状态: 事件`。**约束**：`-->` 两侧必须是**未加引号的状态 ID**，不能写 `"待发布(100)"` 等引号字符串。若需**状态框与箭头都显示中文**，用 `state "待发布(100)" as id`，迁移中只写 id，迁移标签用**中文**（如「开始分发」），否则箭头上会显示英文；避免 `state id : 描述`，部分渲染器只显示 id。有复合状态时用 `state 名 { ... }`。
- **类图/ER 图**：按规范中的关系符号（如 `<|--`、`*--`、`||--o{`）和成员写法使用。

涉及**中文、括号、冒号、问号等**的节点文案或连线文字，一律用**引号包裹**（见 `./reference.md` 中的「特殊字符处理」）。
节点 **ID 用简短英文**（如 `start`、`checkAuth`），禁止中文 ID 和带连字符的 ID。

### 第四步：生成 Mermaid 代码

1. 打开并对照 **`./reference.md`** 中对应类型的示例与「常见错误」小节。
2. 按本技能第二步、第三步的选型与组件规则写出完整代码。
3. 输出时用 Markdown 代码块包裹，注明语言为 `mermaid`：

   ````markdown
   ```mermaid
   graph TD
       start(开始) --> step1["步骤说明(可含括号)"]
       step1 --> step2[结束]
   ```
   ````

4. 在代码块前用 1～2 句话说明：本图类型、表达的重点、若有多图则说明各自用途。

### 第五步：自检（必须执行）

用 `./reference.md` 文末「检查清单」逐项核对：

- 语法正确、可渲染
- 含特殊字符的文本均用引号包裹
- 节点 ID 为简短英文、无连字符
- 中文正常显示
- 类型与场景匹配，节点与连线含义清晰，复杂图考虑拆分或子图

若有违反，修正后再输出。

## 与规范的对应关系

| 本技能步骤     | 对应 reference.md 内容 |
|----------------|-----------------------------|
| 解析任务       | 无直接对应，依赖业务理解    |
| 选择图表类型   | 二～七节：流程图/时序图/类图/状态图/甘特图/ER 图 |
| 选择组件与样式 | 各节的「节点形状」「连接线类型」「参与者」「关系类型」等 |
| 生成语句       | 各节示例代码 + 「特殊字符处理」「节点 ID 规范」「中文文本规范」 |
| 自检           | 第十节「检查清单」          |

## 简要示例（仅作选型参考，完整语法以 reference.md 为准）

**任务**：“画一下用户登录到验证成功的流程。”

- 解析：表达「步骤与分支」，受众为技术/产品，节点数少，单图即可。
- 选型：流程图 `graph TD`。
- 组件：开始(圆角) → 输入/判断/成功/失败(按规范形状) → 结束(圆角)；分支用 `-->|"成功"|` / `-->|"失败"|`。
- 生成：在 reference.md 的流程图示例基础上改写为「登录→验证→判断→成功/失败→结束」，所有含括号/冒号的文案加引号。
- 自检：跑一遍检查清单后输出。
