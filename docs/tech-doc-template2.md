#### 6.2.1 应急方案

针对代码发布（必须）、灰度放量（必须）、常态运行（按需）阶段可能出现的异常，设计高效的应急止损预案。每种应急手段需回答：什么条件下触发？具体操作步骤？预计恢复耗时？是否有数据副作用需要处理？

- **回滚**：回滚顺序是什么？回滚后老代码是否兼容新数据？是否需要数据修复脚本？"直接回滚代码"之类的说法要有理论支撑。
- **限流**：哪些接口需要限流？限流阈值怎么定？触发后用户看到什么（报错/排队/兜底响应）？是否会导致上游超时重试引发雪崩？
- **扩容**：什么水位触发扩容？能自动触发还是需要人工？扩容后下游（DB连接池、缓存、MQ）能否承受？生效需要多久？
- **降级**：哪些功能可降级？开关在哪（Lion/本地配置）？降级后用户体验是什么？是否有数据不一致风险？恢复时需要数据补偿吗？
- **切流/熔断/人工介入**：是否需要切流到备用集群？对下游依赖是否配置了熔断？是否有需要 DBA/SRE 协助的操作？
